# Cron Environment

Cron runs in a minimal shell with almost no environment: no `PATH` beyond `/usr/bin:/bin`, no rbenv or asdf activation, no user-defined variables. Without intervention, `aia` would not be found and jobs would silently fail.

## How aias Solves This

Every cron entry generated by `aias` wraps the `aia` invocation in a **login shell**:

```
0 8 * * * /bin/zsh -l -c 'aia daily_digest >> ~/.aia/schedule/logs/daily_digest.log 2>&1'
```

The `-l` flag sources your shell's login profile (`.zshrc`, `.bash_profile`, etc.), which activates rbenv, loads `PATH` additions, and sets any environment variables your `aia` installation depends on.

## Shell Detection

The shell binary is read from `ENV["SHELL"]` at the time `aias update` is run. If `SHELL` is not set, `/bin/bash` is used as a fallback.

```bash
echo $SHELL
# /bin/zsh
```

The resulting job template:

```ruby
set :job_template, "/bin/zsh -l -c ':job'"
```

## Supported Shells

Any POSIX-compatible shell that supports the `-l` (login) and `-c` (command) flags:

| Shell | Works |
|---|---|
| `bash` | Yes |
| `zsh` | Yes |
| `fish` | Yes (fish supports `-l -c`) |
| `sh` | Yes |

## Troubleshooting Cron Failures

If jobs are failing silently, check the log file:

```bash
cat ~/.aia/schedule/logs/daily_digest.log
```

Common causes:

**`aia` not found:**

```
zsh: command not found: aia
```

Fix: ensure `aia` is installed and `gem install aia` runs inside your login shell. Verify with:

```bash
bash -l -c "which aia"
```

**`AIA_PROMPTS_DIR` not set in cron:**

Cron does not inherit your interactive shell's exports. Set `AIA_PROMPTS_DIR` in your shell's login profile (`.bash_profile`, `.zprofile`), not just in `.bashrc` or `.zshrc` (which are only sourced for interactive shells).

```bash
# ~/.zprofile or ~/.bash_profile
export AIA_PROMPTS_DIR="$HOME/.prompts"
export AIA_API_KEY="sk-..."
```

**rbenv / asdf not activated:**

Add rbenv or asdf initialization to your login profile:

```bash
# ~/.zprofile
eval "$(rbenv init -)"
```

## Environment Variables in Cron Jobs

All environment variables that `aia` needs must be set in your login profile. Cron does not read `.env` files or any project-level configuration.

Required variables (typical):

| Variable | Description |
|---|---|
| `AIA_PROMPTS_DIR` | Path to your prompts directory |
| `ANTHROPIC_API_KEY` or `OPENAI_API_KEY` | API key for the AI backend |

Any variable set in your login profile is available in `aias`-scheduled cron jobs.
